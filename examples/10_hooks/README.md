# 10 hooks

## 场景

有些逻辑不适合写进 prompt，也不适合塞进 tool。

这一章预留给 hooks 示例。它会说明怎么在模型调用、tool 调用或状态更新前后插入自定义逻辑，用来接入记录、权限检查、人工确认或业务规则。

## 关键点

- hook 是运行过程里的扩展点。
- hook 可以观察，也可以改变后续行为。
- 需要说明 hook 的输入、输出和错误处理。
- 不要把核心业务流程拆成一堆隐形 hook。

## 取舍

hooks 适合处理横切逻辑。主流程仍然应该留在清楚的代码、tool 或服务编排里。
