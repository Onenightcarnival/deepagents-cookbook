# 09 service integration

## 场景

agent 最终通常要被应用系统调用，而不是只在命令行里运行。

这一章预留给服务化集成示例。它会说明怎么把一次 agent run 包成服务接口，怎么传入用户信息、业务上下文和 thread id，怎么返回任务状态和错误。

## 关键点

- 设计 request 和 response，避免把内部 state 直接暴露给调用方。
- 说明同步调用、后台任务、轮询状态和流式输出的边界。
- 和持久化配合，让服务重启后还能恢复任务。
- 和可观测配合，让接口错误能回到具体运行步骤。

## 取舍

这一章不写成 Web 框架教程。重点是 agent 和业务系统的边界：输入怎么进来，状态放哪里，错误怎么返回。
